# AI Music Studio — 開発憲章

このプロジェクトは「AI音楽クリエイターが安心して毎日使える制作基地」を作るプロジェクト。
単なるWebアプリではなく、以下の思想を**すべての設計・実装・リファクタリングにおいて最優先**とする。
迷ったら「よりシンプル」「より安全」「より高速」「よりプライバシー重視」を選ぶこと。

## 第一原則: ユーザーの作品はユーザーのもの

歌詞・プロンプト・楽曲・画像・動画・設定・制作履歴・レビューはすべてユーザーの財産。
開発者(および実装するAI)が勝手に取得・収集・解析してはいけない。

## 第二原則: データはユーザーが管理する

- MVPでは**サーバー・DB・Firebase Authentication・Supabase・ユーザー登録・ログイン・クラウド保存を禁止**。
- Cookieへの重要データ保存も禁止。
- すべてのデータはブラウザ内(LocalStorage / IndexedDB)にのみ保存し、外部へ送信しない。
- 開発者が閲覧できる設計にしない。

## 第三原則: いつでも持ち出せる

JSON / Markdown / ZIP でいつでもエクスポートでき、いつでも復元できる。データを人質にする設計は禁止。

## 第四原則: 軽さは正義

新しいライブラリを追加する前に「本当に必要か」を検討する。機能の多さより「気持ちよく動くこと」を優先する。

## 第五原則: 迷わせないUI

機能を探すUIではなく制作が自然に進むUIにする。ボタンは少なく、画面はシンプルに、余白を大切にする。

## 第六原則: 制作の邪魔をしない

通知・広告・ポップアップを乱発しない。ユーザーの集中を最優先する。

## 第七原則: オフラインで完結する

PWA対応。ネット接続がなくても歌詞・プロンプト・メモ・制作管理・AI秘書は利用できる。

## 第八原則: AIは主役ではない

主役はクリエイター。AIは秘書として提案・応援・整理するのみで、勝手に制作しない。最終判断は必ずユーザー。

## 第九原則: カスタマイズは自由

スタジオ・テーマ・AI秘書(画像/名前/一人称/話し方/性格/口癖/応援スタイル)は自由に変更できる。

## 第十原則: 毎日帰ってきたくなる場所

「便利だから使う」ではなく「愛着があるから毎日開く」体験を最優先する。

---

## AI API・AI秘書について

- **MVPではAI APIを一切使用しない**(OpenAI/Claude/Gemini/OpenRouter等へ勝手に接続しない)。APIキー入力画面も不要。
- AI秘書はルールベースで実装する(制作状況・継続日数・曲数などから固定メッセージを選ぶ)。
- 将来AI機能を追加する場合:
  - 開発者のAPIキーを使う場合は **Cloudflare Workers / Firebase Functions などサーバーレスのみ**。APIキーはサーバー側にのみ保存し、ブラウザへは絶対に送らない。
  - ユーザー自身のAPIキーに対応する場合は、**暗号化してローカル保存**し、開発者のAPIキーは使わない。

## デプロイ・技術方針

- **GitHub Pagesへそのままデプロイできる静的サイト構成**を維持する。Node/PHP/Python/Dockerサーバーは不要。
- ルーティングは HashRouter(または同等の静的ホスティング互換の方式)を使う。
- ビルド成果物(`dist/`)だけで完結すること。

## 関連する外部ツール(同じ制作者が運営する姉妹ツール)

これらは別ドメインで独立稼働している静的ツール。本アプリはこれらへの導線(リンク)を提供する。中身を再実装しない。

- マスタリング自動生成ツール: https://karasui1014.github.io/lofi-detective-website/mastering/
- スタイルプロンプト工房: https://karasui1014.github.io/lofi-detective-website/style-prompt-koubou/

## Claudeへの最終指示

実装中に「もっと便利だから」という理由で、ユーザーデータの収集・クラウド保存・強制ログイン・外部サービスへの送信・不要なライブラリ追加・不要な依存関係を行わないこと。
